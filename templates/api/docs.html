Introduction

Cette API permet de gérer les opérations de la mutuelle de santé, incluant la gestion des soins, des membres, des paiements et des statistiques.

Base URL: http://votre-domaine.com/api/
Format des données: JSON
Encodage: UTF-8

Authentification

POST /api/token/

Authentification avec token DRF standard.

Requête: 
{
  "username": "votre_utilisateur",
  "password": "votre_mot_de_passe"
}
Réponse (200): 
{
  "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b"
}

Utilisation: Ajouter le header Authorization: Token {votre_token} aux requêtes suivantes.

POST /api/auth/token/

Authentification avec JSON Web Tokens (JWT).

Requête: 
{
  "username": "votre_utilisateur",
  "password": "votre_mot_de_passe"
}
Réponse (200): 
{
  "refresh": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "access": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9..."
}
Refresh Token: Utiliser /api/auth/token/refresh/ pour rafraîchir le token d'accès.

Gestion des Soins

GET /api/soins/

Récupère la liste des soins avec pagination et filtres.

Paramètres GET
Query
page - Numéro de page (défaut: 1)
page_size - Taille de page (défaut: 20)
search - Recherche textuelle
ordering - Tri (ex: date_realisation)
statut - Filtre par statut
Réponse (200): 
{
  "count": 42,
  "next": "http://api.example.com/api/soins/?page=2",
  "previous": null,
  "results": [...]
}

POST /api/soins/

Crée un nouveau soin (réservé aux médecins).

patient
Integer
ID du patient Requis
type_soin
Integer
ID du type de soin Requis
date_realisation
Date
Date de réalisation (YYYY-MM-DD) Requis
Requête: 
{
  "patient": 1,
  "type_soin": 3,
  "date_realisation": "2024-01-15",
  "cout": 150.00,
  "description": "Consultation générale",
  "diagnostic": "Grippe saisonnière"
}

Vérification de Santé

GET /api/health/

Vérifie si l'API est fonctionnelle. Ne nécessite pas d'authentification.

Réponse (200): 
{
  "status": "healthy",
  "service": "Mutuelle Core API",
  "version": "1.0",
  "timestamp": "2024-01-15T10:30:00Z",
  "environment": "production"
}

Statistiques

GET /api/statistiques/

Récupère les statistiques selon le rôle de l'utilisateur.

Note: Les statistiques varient selon le rôle :
  • Assureur: Statistiques globales
  • Médecin: Statistiques personnelles
  • Membre: Statistiques personnelles
